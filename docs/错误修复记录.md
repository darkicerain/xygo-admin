# 系统配置页面 - 错误修复记录

## 已修复的错误

### 1. ✅ vuedraggable 模块解析失败
**错误信息**:
```
Failed to resolve import "vuedraggable" from "src/components/core/forms/art-image-upload/index.vue"
```

**原因**: 项目使用 `vue-draggable-plus` 而不是 `vuedraggable`

**修复**:
```typescript
// 修复前
import draggable from 'vuedraggable'
<draggable v-model="imageList">
  <template #item="{ element }">...</template>
</draggable>

// 修复后
import { VueDraggable } from 'vue-draggable-plus'
<VueDraggable v-model="imageList">
  <div v-for="(element, index) in imageList" :key="element.id">...</div>
</VueDraggable>
```

### 2. ✅ 类型检查错误 - switch/checkbox
**错误信息**:
```
此比较似乎是无意的，因为类型"string"和"number"没有重叠
```

**原因**: 直接比较 string 与 number/boolean

**修复**:
```typescript
// 修复前
formData[schema.key] = (value === '1' || value === 1 || value === true) ? '1' : '0'

// 修复后
const strValue = String(value || '0')
formData[schema.key] = (strValue === '1' || strValue === 'true') ? '1' : '0'
```

---

### 3. ✅ ElInputNumber 类型错误
**错误信息**:
```
Invalid prop: type check failed for prop "modelValue". Expected Number | Null, got String with value "465"
```

**原因**: 数字类型的值是字符串

**修复**:
```typescript
case 'number':
  // 转换为数字，空值设为 null
  if (value === '' || value === null || value === undefined) {
    formData[schema.key] = null
  } else {
    const num = Number(value)
    formData[schema.key] = isNaN(num) ? null : num
  }
  break
```

---

### 4. ✅ ArtWangEditor undefined 错误
**错误信息**:
```
Invalid prop: type check failed for prop "modelValue". Expected String with value "undefined", got Undefined null
```

**原因**: 
1. editor 类型字段可能未初始化
2. ArtWangEditor 使用 `defineModel<string>({ required: true })`，不接受 undefined

**修复**:
```typescript
// 1. 数据初始化时确保字符串类型
case 'textarea':
case 'editor':
  formData[schema.key] = value !== null && value !== undefined ? String(value) : ''
  break

// 2. 添加渲染守卫
<ArtWangEditor
  v-else-if="schema.type === 'editor' && configFormData[schema.key] !== undefined"
  v-model="configFormData[schema.key]"
/>
<ElSkeleton v-else-if="schema.type === 'editor'" :rows="3" animated />

// 3. 确保 schema 加载完成后再加载数据
if (Object.keys(configSchemaMap.value).length === 0) {
  await loadConfigSchema()
}
```

### 5. ✅ Props 默认值问题
**错误信息**:
```
Expected String with value "undefined", got Undefined
```

**原因**: Props 中设置了 `modelValue: ''` 的默认值，但 Vue 3 中可选属性不应该设置默认值

**修复**:
```typescript
// 修复前
const props = withDefaults(defineProps<Props>(), {
  modelValue: '',  // ❌ 不要为可选的 modelValue 设置默认值
  multiple: false
})

// 修复后
const props = withDefaults(defineProps<Props>(), {
  // modelValue 不设默认值 ✅
  multiple: false
})

// 在 computed 中处理 undefined
const colorValue = computed({
  get: () => props.modelValue || '',  // ✅ 确保返回空字符串
  set: (val) => emit('update:modelValue', val || '')
})
```

## 💡 经验总结

### Vue 3 Props 最佳实践

1. **可选的 modelValue 不要设默认值**
```typescript
// ❌ 错误
interface Props {
  modelValue?: string
}
const props = withDefaults(defineProps<Props>(), {
  modelValue: ''  // 不要这样做
})

// ✅ 正确
const props = withDefaults(defineProps<Props>(), {
  // modelValue 不设默认值
})
```

2. **在 computed 中处理 undefined**
```typescript
const value = computed({
  get: () => props.modelValue || '',  // 确保返回有效值
  set: (val) => emit('update:modelValue', val || '')
})
```

### 类型转换最佳实践

1. **避免跨类型比较**
```typescript
// ❌ 错误
if (value === 1 || value === true)  // value 是 string

// ✅ 正确
const strValue = String(value)
if (strValue === '1' || strValue === 'true')
```

2. **数字类型转换**
```typescript
// ✅ 安全的数字转换
const num = Number(value)
formData.key = isNaN(num) ? null : num
```

3. **为每种类型提供合法默认值**
```typescript
switch (type) {
  case 'number': return null  // ✅ 数字用 null
  case 'switch': return '0'   // ✅ 开关用 '0'
  case 'array': return []     // ✅ 数组用 []
  case 'json': return '{}'    // ✅ 对象用 '{}'
  default: return ''          // ✅ 默认用空字符串
}
```

### 组件渲染最佳实践

1. **添加渲染守卫**
```vue
<!-- 对于 required 的 props，添加条件判断 -->
<ArtWangEditor
  v-if="schema.type === 'editor' && configFormData[schema.key] !== undefined"
  v-model="configFormData[schema.key]"
/>
<ElSkeleton v-else-if="schema.type === 'editor'" :rows="3" />
```

2. **确保数据准备完成**
```vue
<!-- 等待数据加载完成再渲染 -->
<div v-if="dataReady && items.length > 0">
  <!-- 渲染内容 -->
</div>
```

## ⚠️ 常见错误说明

### ResizeObserver 警告
**错误信息**: `ResizeObserver loop completed with undelivered notifications`

**说明**: 这不是错误，是浏览器的正常警告，通常发生在：
- 大量 DOM 元素同时更新
- 使用了响应式布局组件（ElDialog、ElDrawer等）
- 可以安全忽略
