# 系统配置开发文档（方案与步骤）

## 1. 目标与原则
- 动态化：配置项由数据库驱动，可新增/删除/排序/分组。
- 类型安全：存储为字符串/JSON，服务层做强类型校验与转换。
- 性能：读优先走缓存（gcache/Redis），写入刷新缓存，避免频繁查库。
- 扩展：同一套元数据驱动前端渲染不同组件。

## 2. 数据模型设计
表：`xy_sys_config`（单表，存定义 + 当前值）
- `id`
- `group`(分组标识，如 basics/mail/oss)、`group_name`(中文名)
- `name`(显示名)、`key`(唯一键)
- `value` text（存字符串/JSON）
- `type` 控件类型：`text/textarea/number/switch/select/radio/checkbox/json/object/array/color/upload`
- `options` json：组件参数/选项，如 `{ "options":[{"label":"是","value":"1"}], "placeholder":"", "min":0 }`
- `rules` json：校验规则，如 `{ "required":true, "pattern":"^[a-zA-Z0-9_]{2,}$", "min":1, "max":50 }`
- `sort`、`remark`、`created_at/updated_at`

## 3. 缓存与读取策略
- 缓存 Key：`sys:config:group:{group}`，存 `map[string]string`。
- 读取：Cache → DB（命中后回写 Cache）。
- 写入：事务写 DB → 刷新/淘汰对应组缓存。
- 预热：启动可选全量预热；也可按组懒加载。

## 4. 后端接口（管理端）
- `GET  /admin/config/schema`：返回全部分组的配置项定义（驱动前端渲染）。
- `GET  /admin/config/list?group=basics`：返回该组 `key→value`。
- `POST /admin/config/save`：`{ group, items:[{key,value}] }`，按 type 校验后批量保存并刷新缓存。
- 可选：`POST /admin/config/item/save`、`/delete`（高级模式动态增删配置项）。

## 5. Service 设计
- `ConfigService.Get(ctx, key, outPtr)`：按 key 取值并按 item.type 转换。
- `ConfigService.GetGroup(ctx, group) map[string]string`
- `ConfigService.SetGroup(ctx, group, kv map[string]any)`：保存后刷新缓存。
- `ConfigService.WarmUp(ctx)`：启动预热。
校验与转换：
- 保存：按 type 做校验与规范化（number→string；switch→"0/1"；json/object/array 先 decode 再存字符串）。
- 读取：提供 ToString/ToBool/ToInt/ToJSON 便捷方法。

## 6. 前端渲染思路
- 获取 `schema` → 按 `group` 渲染 Tab。
- 根据 item.type 选择组件：text/textarea/number/switch/select/radio/checkbox/json/object/array/color/upload…
- 选项/placeholder/校验规则来自 item.options/item.rules。
- 提交仅发 `{key,value}`；不携带展示属性。
- “新增配置项”入口放高级模式，对接 item.save。
- 临时静态菜单新增「系统设置」入口（动态菜单完成后切换为后端路由）。

## 7. 权限与安全
- 菜单：新增系统设置页入口。
- 按钮权限：如 `PUT /admin/config/save`，在后续接口权限中按 `METHOD+PATH` 校验。
- 仅高权限角色可见（R_SUPER / R_ADMIN）。

## 8. 实施步骤（建议顺序）
1) **建表**：`xy_sys_config_item`、`xy_sys_config_value`，插入基础分组与示例项（站点名/备案号/时区/禁止 IP/SMTP 等）。
2) **Service**：实现读取/保存/预热、类型校验与转换、缓存读写。
3) **接口**：实现 `/admin/config/schema`、`/list`、`/save`，接入权限。
4) **前端页面**：新增系统设置页，Tab+动态表单渲染，接入保存。
5) **静态菜单**：加「系统设置」入口；待动态菜单接入后改为后端驱动。
6) **验收**：配置增删改查、缓存命中、类型校验、权限控制。

## 9. 验收要点
- 可通过接口获取 schema 和某分组配置值，保存后缓存被刷新。
- 类型校验有效（如 number/switch/json/object/array 不合法时拒绝）。
- 系统设置页可渲染不同控件类型，保存后刷新再读取值一致。
- 权限：非授权角色无法访问保存接口。

## 10. 后续增强
- 配置项导入/导出（JSON/CSV）以便环境迁移。
- 变更审计（who/when/old/new）。
- 可选多租户/多环境隔离策略（按 tenant/env 分桶）。
