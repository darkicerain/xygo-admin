
**通知推送测试**（topic: `notice_push`）：
```json
{

  "event": "notice/new",
  "payload": {"title": "测试通知", "content": "这是一条来自消息队列的通知"}

```

### 2. 后端代码测试

在任意 Go 文件中：

```go
import "xygo/internal/library/queue"

// 投递一条消息

    "user_id":    1,
    "module":     "test",

```

## 二、新增消费者（3 步）

### 步骤 1：创建消费者文件

在 `server/internal/queues/` 新建文件，例如 `email_send.go`：

```go
package queues

import (
    "context"
    "encoding/json"

    "github.com/gogf/gf/v2/frame/g"
    "xygo/internal/library/queue"
)

// Topic 常量，建议统一定义在这里
const TopicEmailSend = "email_send"

func init() {
    queue.Register(&EmailSendConsumer{})
}

type EmailSendConsumer struct{}

func (c *EmailSendConsumer) GetTopic() string { return TopicEmailSend }

func (c *EmailSendConsumer) Handle(ctx context.Context, msg *queue.Message) error {
    var data struct {
        To      string `json:"to"`
        Subject string `json:"subject"`
        Body    string `json:"body"`
    }
    if err := json.Unmarshal([]byte(msg.Body), &data); err != nil {
        g.Log().Errorf(ctx, "[queue:email_send] unmarshal failed: %v", err)
        return nil // 格式错误不重试
    }

    // TODO: 调用邮件发送 SDK
    g.Log().Infof(ctx, "[queue:email_send] sending to %s: %s", data.To, data.Subject)

    return nil // 返回 nil 表示消费成功
}
```

### 步骤 2：无需额外注册

`init()` 中的 `queue.Register()` 会自动注册。`main.go` 已有 `_ "xygo/internal/queues"` 导入，新文件自动生效。

### 步骤 3：在业务中投递

```go
import "xygo/internal/library/queue"

// 在需要发邮件的地方
queue.Push("email_send", map[string]interface{}{
    "to":      "user@example.com",
    "subject": "欢迎注册",
    "body":    "您好，欢迎使用 XYGo Admin！",
})
```

重启服务后，消费者自动开始监听，前端管理页也能看到新的 Topic。

## 三、生产者 API 说明

### `queue.Push(topic, body)`

| 参数 | 类型 | 说明 |
|------|------|------|
| topic | string | 队列主题，与消费者 GetTopic() 对应 |
| body | any | 消息内容，支持 string / []byte / struct / map（自动 JSON 序列化） |

```go
// 字符串
queue.Push("my_topic", `{"key":"value"}`)

// 结构体
queue.Push("my_topic", MyStruct{Name: "test"})

// Map
queue.Push("my_topic", g.Map{"name": "test", "age": 18})
```

### 消息自动能力

| 能力 | 说明 |
|------|------|
| **自动重试** | 消费者返回 error 时自动重新入队，最多 3 次 |
| **死信队列** | 重试耗尽后消息进入 `{topic}:dead`，不会丢失 |
| **并发消费** | 每个 Topic 一个 goroutine 监听，互不阻塞 |
| **优雅停机** | 服务关闭时自动停止所有消费者 |

## 五、驱动配置

在 `manifest/config/config.yaml` 中添加（可选，默认用 Redis）：

```yaml
# 消息队列配置
queue:
  driver: "redis"       # redis 或 disk
  diskPath: "storage/queue"  # disk 驱动的存储路径
```

| 驱动 | 适用场景 | 特点 |
|------|---------|------|
| **redis**（默认） | 生产环境 | 高性能、支持分布式、BRPOP 阻塞弹出 |
| **disk** | 开发/无 Redis 环境 | 零依赖、文件存储、单机使用 |

## 六、常见场景示例

### 场景 1：用户注册后发欢迎邮件

```go
// controller 层
func (c *ControllerV1) Register(ctx context.Context, req *api.RegisterReq) (...) {
    // ... 注册逻辑 ...

    // 异步发邮件（不阻塞注册响应）
    queue.Push("email_send", g.Map{
        "to":      req.Email,
        "subject": "欢迎注册",
        "body":    "您好 " + req.Username,
    })
}
```

### 场景 2：订单超时自动取消

```go
// 创建消费者 queues/order_cancel.go
const TopicOrderCancel = "order_cancel"

func init() { queue.Register(&OrderCancelConsumer{}) }

type OrderCancelConsumer struct{}
func (c *OrderCancelConsumer) GetTopic() string { return TopicOrderCancel }
func (c *OrderCancelConsumer) Handle(ctx context.Context, msg *queue.Message) error {
    // 检查订单状态，未支付则取消
    return nil
}

// 在定时任务中扫描超时订单并投递
queue.Push("order_cancel", g.Map{"orderId": 12345})
```

### 场景 3：批量数据导入

```go
// 拆分成多条小消息投递
for _, row := range bigDataList {
    queue.Push("data_import", row)
}
// 消费者逐条处理，失败自动重试，不会丢数据
```

## 七、监控与排查

### 查看队列积压

前端页面"系统管理 → 消息队列"可直接查看各 Topic 的待消费数和死信数。

### 日志关键字

```
[queue] initialized          — 队列初始化完成
[queue] consumer listening   — 消费者开始监听
[queue] consumer handle failed — 消费失败（会重试）
[queue] message re-queued    — 消息重新入队
[queue] message moved to dead letter — 进入死信队列
```

### Redis 直接查看

```bash
# 查看队列积压
redis-cli LLEN xygo:queue:operation_log

# 查看死信队列
redis-cli LLEN xygo:queue:operation_log:dead

# 查看队列内容（不消费）
redis-cli LRANGE xygo:queue:operation_log 0 -1
```
