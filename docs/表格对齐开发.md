# 表格能力对齐方案（对标 BuildAdmin）

## 1. 差异概览
- BuildAdmin（参考 `buildadmin-2/web/src/views/backend/user/user/index.vue`）提供声明式的列/表单元数据：`render`（datetime/tag/image/buttons）、`operator`（=、LIKE、RANGE、自定义排序）、`replaceValue/custom`、远程下拉、批量按钮、列显示/快捷搜索/高级搜索等全部内置。
- 现有框架（`useTable` + `ArtTable` + 手写 formatter/form）主要支持数据获取、分页、列勾选/显示、缓存、防抖，但列渲染/搜索/排序/表单类型基本依赖手写 formatter/slot。

## 2. BuildAdmin 有而现有缺/弱的能力
1) 列渲染类型（当前没有语义化内置，需要新增）  
   - `render: 'datetime'`：时间自动格式化显示。  
   - `render: 'tag'`：按值渲染标签并支持颜色映射。  
   - `render: 'image'`：图片缩略图 + 预览。  
   - `render: 'buttons'`：行操作按钮组。  
   - `replaceValue`：值 → 显示文案映射。  
   - `custom`：值 → 颜色/类型映射（如标签颜色）。 
2) 查询/搜索配置  
   - `operator` 声明（=、LIKE、RANGE、自定义排序），自动生成查询表单和请求参数。  
   - `quickSearch` / `comSearch` 入口与默认占位。  
3) 排序/分页  
   - `sortable: 'custom'` 自动带上排序字段与方向。  
4) 列/表格交互  
   - `columnDisplay`（列显隐 UI）、`selection/index/drag` 等快速开关。  
   - 顶部批量按钮集成（refresh/add/edit/delete）。  
5) 表单层（PopupForm）  
   - 统一 `FormItem`，支持 `remoteSelect`、`image` 上传、`radio/checkbox`、`date/datetime`、占位/校验/远程加载等配置化。  
   - `defaultItems` 初始值、规则集中配置，密码等特殊校验内置。  
6) 远程/字典  
   - 下拉远程加载、字典映射内置在配置项里，不需手写异步逻辑。  

## 3. 建议改造方案（分阶段，尽量减小侵入）
### Phase A（最小可用，1~2 天）
- 新增“列渲染器”适配层：在 `ArtTable` 外包一层 `SchemaTable`（或在现有 `columnsFactory` 里支持 `type` 字段），内置 type → 渲染器映射：`text/default`、`tag/badge`、`image`、`datetime`、`buttons`。保留 formatter 作为兜底。
- 补充格式化工具：统一时间格式（毫秒/秒自动识别，默认 `YYYY-MM-DD HH:mm:ss`），标签颜色映射。
- 兼容策略：旧列无 type 时按 text 渲染；已有 formatter 优先。

### Phase B（搜索/排序声明式，1~1.5 天）
- 列配置增加 `search` 描述：`{ enable: true, operator: 'like'|'='|'range'|'in', component: 'input'|'select'|'daterange', options?, remote? }`。在表格头/独立搜索区自动生成搜索表单，提交时转换为 API 参数。  
- 支持 `sortable: 'custom'`：触发时将字段/方向注入请求参数（兼容后端排序字段名约定）。

### Phase C（表单配置化，1~1.5 天）
- 抽象 `FormItem` 组件：支持 input/textarea/select/remoteSelect/radio/checkbox/date/datetime/range/upload。  
- 表单 schema 支持：占位、校验规则、字典/远程、默认值、禁用/只读、tips。  
- 提供“表单驱动”示例（先落在用户弹窗），保持手写模式兼容。

### Phase D（增强交互，0.5~1 天）
- 列显隐/顺序 UI 强化：现有列勾选基础上，补充拖拽排序与持久化。  
- 顶部批量按钮集成：包装一层默认按钮组（刷新/新增/编辑/删除），可插槽扩展。

## 4. 任务拆解与落地文件
- 新增/改动文件建议：
  - `web/src/components/schema-table/`：SchemaTable 主体 + 渲染器（TagCell/ImageCell/DatetimeCell/ButtonCell）。  
  - `web/src/utils/table/format.ts`：时间/字典/颜色格式化。  
  - `web/src/hooks/core/useTable`：扩展排序/搜索参数适配（保持向后兼容）。  
  - `web/src/components/schema-form/`（可选阶段 C）：FormItem 及适配器。  
- 试点页面：`web/src/views/system/user/index.vue` 先接入 SchemaTable，验证列渲染和搜索。

## 5. 兼容与风险
- 兼容：旧列配置与 formatter/slot 保持可用；新字段 type/search/sortable 增量可选。  
- 后端契约：需统一排序/筛选参数的字段名（建议 current/size/sortField/sortOrder；range 传开始结束）。  
- 远程/字典：需要约定 options 数据源（常量/接口）；远程选择要有防抖和 loading。
