# 岗位/部门模块部署步骤

## 1. 执行岗位表SQL

```bash
# 在MySQL中执行
mysql -u root -p xygo < docs/岗位管理-数据库脚本.sql
```

或者直接在数据库客户端执行 `docs/岗位管理-数据库脚本.sql` 中的SQL。

## 2. 生成DAO代码

```bash
cd server
gf gen dao
```

这会自动生成：
- `internal/dao/admin_post.go`
- `internal/dao/admin_user_post.go`
- `internal/model/entity/admin_post.go`
- `internal/model/entity/admin_user_post.go`
- `internal/model/do/admin_post.go`
- `internal/model/do/admin_user_post.go`

## 3. 重新编译

```bash
cd server
go build
```

## 4. 启动服务

```bash
cd server
go run main.go
```

## 当前状态

- ✅ 后端代码：已创建（Controller、API、Input）
- ✅ 前端代码：已创建（页面、组件、API调用）
- ⚠️ 数据库表：**需要执行SQL**
- ⚠️ DAO代码：**需要运行 gf gen dao**

## 注意

在执行 `gf gen dao` 之前，确保：
1. 数据库配置正确（`server/manifest/config/config.yaml`）
2. 数据库中已创建岗位表
3. 表名前缀为 `xy_`（与项目配置一致）
