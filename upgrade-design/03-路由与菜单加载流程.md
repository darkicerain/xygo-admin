# 路由与菜单加载流程（BuildAdmin 风格）

## 1. 整体流程

```
用户访问
    ↓
路由守卫 beforeEach
    ↓
判断登录状态
    ↓ (已登录)
获取用户信息
    ↓
获取菜单数据（包含插件菜单）
    ↓
动态注册路由
    ↓
权限验证
    ↓
页面渲染
```

## 2. 路由组织结构

```
src/router/
├── index.ts               # 路由主入口
├── guards/                # 路由守卫
│   ├── auth.ts            # 认证守卫
│   └── permission.ts      # 权限守卫
├── static.ts              # 静态路由（登录、404 等）
├── backend/               # 后台路由模块
│   ├── index.ts           # 后台路由汇总
│   ├── system.ts          # 系统管理路由
│   ├── dashboard.ts       # 仪表盘路由
│   └── [plugin].ts        # 插件路由（安装后生成）
└── frontend/              # 前台路由模块
    └── index.ts
```

## 3. 静态路由 vs 动态路由

### 静态路由
```typescript
// src/router/static.ts
export const staticRoutes: RouteRecordRaw[] = [
  {
    path: '/login',
    name: 'Login',
    component: () => import('@/views/backend/auth/login.vue')
  },
  {
    path: '/404',
    name: 'NotFound',
    component: () => import('@/views/common/404.vue')
  }
]
```

### 动态路由（基于菜单）
```typescript
// src/router/backend/system.ts
export default [
  {
    path: '/system/user',
    name: 'SystemUser',
    component: () => import('@/views/backend/system/user/index.vue'),

    path: '/system/role',
    name: 'SystemRole',
    component: () => import('@/views/backend/system/role/index.vue'),
    meta: { title: '角色管理', perms: ['system:role:view'] }
  }

```

## 4. 菜单加载流程

```typescript
// 1. 用户登录后获取菜单
const menuStore = useMenuStore()
await menuStore.fetchMenus()

// 2. menuStore.fetchMenus 实现
async fetchMenus() {
  // 从后端获取菜单（已包含插件菜单）
  const res = await adminRequest.get('/admin/menu/tree')
  this.menus = res.data
  
  // 生成动态路由
  const routes = generateRoutes(this.menus)
  
  // 注册到路由
  routes.forEach(route => {
    router.addRoute('backend', route)
  })
}
```

## 5. 路由守卫实现

```typescript
// src/router/guards/auth.ts
router.beforeEach(async (to, from, next) => {
  const userStore = useUserStore()
  const menuStore = useMenuStore()
  
  // 白名单路由直接放行
  if (whiteList.includes(to.path)) {
    return next()
  }
  
  // 未登录跳转登录页
  if (!userStore.token) {
    return next(`/login?redirect=${to.path}`)
  }
  
  // 已登录但未加载菜单
  if (!menuStore.isLoaded) {
    try {
      // 获取用户信息
      await userStore.fetchUserInfo()
      
      // 获取菜单并注册路由
      await menuStore.fetchMenus()
      
      // 重新导航到目标路由
      return next({ ...to, replace: true })
    } catch (error) {
      // 获取失败，清除登录状态
      userStore.logout()
      return next('/login')
    }
  }
  
  next()
})
```

## 6. 权限验证

```typescript
// src/router/guards/permission.ts
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 获取路由需要的权限
  const requiredPerms = to.meta.perms as string[] | undefined
  
  if (!requiredPerms || requiredPerms.length === 0) {
    return next()
  }
  
  // 检查用户是否有权限
  const hasPermission = requiredPerms.some(perm => 
    userStore.permissions.includes(perm)
  )
  
  if (hasPermission) {
    next()
  } else {
    next('/403')
  }
})
```

## 7. 插件路由加载

插件安装后，路由文件已复制到 `src/router/backend/[plugin].ts`。

两种加载方式：

### 方式 1：静态导入（构建时确定）
```typescript
// src/router/backend/index.ts
import systemRoutes from './system'
import wechatRoutes from './wechat'  // 插件路由

export default [
  ...systemRoutes,
  ...wechatRoutes
]
```

### 方式 2：动态导入（运行时确定）
```typescript
// 基于菜单数据动态生成路由
function generateRoutes(menus: MenuItem[]): RouteRecordRaw[] {
  return menus.map(menu => ({
    path: menu.path,
    name: menu.name,
    component: () => import(`@/views/backend/${menu.component}.vue`),
    meta: { title: menu.title, icon: menu.icon }
  }))
}
```

## 9. 刷新与状态恢复

```typescript
// 页面刷新时
router.beforeEach(async (to, from, next) => {
  // 检查是否需要重新加载
  if (userStore.token && !menuStore.isLoaded) {
    await menuStore.fetchMenus()
  }
  next()
})
```

## 10. 注意事项

1. **避免重复注册**：使用标志位防止路由重复注册
2. **404 处理**：动态路由加载后再添加 404 通配路由
3. **权限缓存**：权限数据应缓存，避免重复请求
4. **插件热更新**：开发模式下插件路由修改需支持 HMR
