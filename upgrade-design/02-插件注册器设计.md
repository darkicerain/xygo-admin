# 插件管理器设计（PluginManager）

> BuildAdmin 风格：插件独立存放，安装时复制到项目目录

## 1. 设计目标

- 插件独立打包，存放在 `src/modules/` 目录
- 安装时将插件代码复制到项目对应位置
- 卸载时删除代码并备份
- 支持启用/禁用状态切换
- 与后端插件表同步状态

## 2. 插件包结构

```
src/modules/wechat/
├── info.json              # 插件信息（必须）
```

## 4. PluginManager 核心实现

```typescript
// src/utils/plugin/PluginManager.ts

interface PluginInfo {
  id: string
  name: string
  version: string
  description: string
  author: string
  status: 'installed' | 'enabled' | 'disabled' | 'uninstalled'
  menus: MenuItem[]
  permissions: string[]
}

class PluginManager {
  private plugins: Map<string, PluginInfo> = new Map()
  
  /**
   * 获取所有插件
   */
  async getPlugins(): Promise<PluginInfo[]> {
    // 从后端 API 获取插件列表
    const res = await adminRequest.get('/admin/plugin/list')
    return res.data
  }
  
  /**
   * 安装插件
   * 注意：文件复制操作需要后端支持
   */
  async install(pluginId: string): Promise<void> {
    await adminRequest.post('/admin/plugin/install', { pluginId })
  }
  
  /**
   * 卸载插件
   */
  async uninstall(pluginId: string): Promise<void> {
    await adminRequest.post('/admin/plugin/uninstall', { pluginId })
  }
  
  /**
   * 启用插件
   */
  async enable(pluginId: string): Promise<void> {
    await adminRequest.post('/admin/plugin/enable', { pluginId })
  }
  
  /**
   * 禁用插件
   */
  async disable(pluginId: string): Promise<void> {
    await adminRequest.post('/admin/plugin/disable', { pluginId })
  }
  
  /**
   * 获取已启用插件的路由
   */
  async getEnabledRoutes(): Promise<RouteRecordRaw[]> {
    const plugins = await this.getPlugins()
    const enabledPlugins = plugins.filter(p => p.status === 'enabled')
    
    const routes: RouteRecordRaw[] = []
    for (const plugin of enabledPlugins) {
      // 动态导入插件路由
      try {
        const routeModule = await import(`@/router/backend/${plugin.id}.ts`)
        routes.push(...routeModule.default)
      } catch (e) {
        console.warn(`Plugin ${plugin.id} route not found`)
      }
    }
    
    return routes
  }
  
  /**
   * 获取已启用插件的菜单
   */
  async getEnabledMenus(): Promise<MenuItem[]> {
    const plugins = await this.getPlugins()
    const enabledPlugins = plugins.filter(p => p.status === 'enabled')
    
    return enabledPlugins.flatMap(p => p.menus)
  }
}

export const pluginManager = new PluginManager()
export default PluginManager
```

## 5. 后端 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /admin/plugin/list | 获取插件列表 |
| POST | /admin/plugin/install | 安装插件 |
| POST | /admin/plugin/uninstall | 卸载插件 |
| POST | /admin/plugin/enable | 启用插件 |
| POST | /admin/plugin/disable | 禁用插件 |
| POST | /admin/plugin/upgrade | 升级插件 |
| GET | /admin/plugin/market | 获取市场插件 |

## 6. 插件安装流程（后端实现）

```
1. 接收安装请求
2. 从市场下载插件包（或从本地 modules 读取）
3. 解压到 src/modules/[plugin-id]/
4. 读取 info.json
5. 复制文件到对应目录：
   - api/ → src/api/backend/[plugin-id]/
   - views/ → src/views/backend/[plugin-id]/
   - routes/ → src/router/backend/[plugin-id].ts
   - store/ → src/stores/backend/[plugin-id].ts
6. 执行 install.sql（如有）
7. 将菜单写入数据库
8. 将权限写入数据库
9. 更新插件状态为 installed
```

## 7. 插件卸载流程（后端实现）

```
1. 接收卸载请求
2. 备份插件文件到 src/modules/backups/[plugin-id]/
3. 删除已安装的文件：
   - src/api/backend/[plugin-id]/
   - src/views/backend/[plugin-id]/
   - src/router/backend/[plugin-id].ts
   - src/stores/backend/[plugin-id].ts
4. 执行 uninstall.sql（如有）
5. 删除数据库中的菜单
6. 删除数据库中的权限
7. 更新插件状态为 uninstalled
```

## 8. 与路由系统集成

```typescript
// src/router/index.ts

import { pluginManager } from '@/utils/plugin'

// 加载插件路由
async function loadPluginRoutes() {
  const pluginRoutes = await pluginManager.getEnabledRoutes()
  
  for (const route of pluginRoutes) {
    router.addRoute('backend', route)
  }
}
```

## 9. 与菜单系统集成

插件菜单在安装时已写入数据库，后端 `/admin/menu/tree` 返回时会包含插件菜单。

前端无需特殊处理，现有菜单加载逻辑即可工作。

## 10. 开发模式支持

开发时可直接在目标目录创建插件文件，无需通过安装流程：

```
src/api/backend/wechat/
src/views/backend/wechat/
src/router/backend/wechat.ts
```

发布时再打包为插件包格式。
