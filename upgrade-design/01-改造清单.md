# 改造清单（BuildAdmin 风格）

> 按照 BuildAdmin 的目录结构进行改造

## 1. 目录结构调整

### 1.1 API 目录

```
当前：src/api/
目标：
  src/api/
    ├── backend/           # 后台 API
    │   ├── system/        # 系统管理（用户、角色、菜单等）
    │   ├── auth/          # 认证（登录、登出、验证码）
    │   └── common/        # 公共（上传、字典等）
    └── frontend/          # 前台 API（预留）
```

改造步骤：
- [ ] 创建 `src/api/backend/` 目录
- [ ] 创建 `src/api/frontend/` 目录
- [ ] 迁移现有 API 到 `backend/` 对应子目录
- [ ] 更新所有 API 导入路径

### 1.2 Views 目录

```
当前：src/views/
目标：
  src/views/
    ├── backend/           # 后台页面
    │   ├── system/        # 系统管理
    │   ├── auth/          # 认证页面
    │   └── dashboard/     # 仪表盘
    ├── frontend/          # 前台页面（预留）
    └── common/            # 公共页面（错误页等）
```

改造步骤：
- [ ] 创建 `src/views/backend/` 目录
- [ ] 创建 `src/views/frontend/` 目录
- [ ] 迁移系统页面到 `backend/system/`
- [ ] 迁移登录等页面到 `backend/auth/`
- [ ] 更新路由组件路径

### 1.3 Router 目录

```
当前：src/router/
目标：
  src/router/
    ├── index.ts           # 主入口
    ├── guards/            # 路由守卫
    ├── backend/           # 后台路由模块
    │   ├── system.ts      # 系统管理路由
    │   └── index.ts       # 后台路由汇总
    └── frontend/          # 前台路由模块
```

改造步骤：
- [ ] 创建 `src/router/backend/` 目录
- [ ] 创建 `src/router/frontend/` 目录
- [ ] 拆分现有路由到对应目录
- [ ] 更新路由加载逻辑

### 1.4 Store 目录

```
当前：src/stores/
目标：
  src/stores/
    ├── index.ts           # Pinia 入口
    ├── backend/           # 后台状态
    │   ├── user.ts
    │   ├── menu.ts
    │   └── setting.ts
    └── frontend/          # 前台状态
```

改造步骤：
- [ ] 创建 `src/stores/backend/` 目录
- [ ] 创建 `src/stores/frontend/` 目录
- [ ] 迁移现有 store 到 `backend/`
- [ ] 更新 store 导入路径

### 1.5 Modules 目录（插件目录）

```
新增：
  src/modules/             # 插件包存放目录
    ├── .gitkeep
    └── backups/           # 卸载备份
```

## 2. Request 封装改造

```typescript
// src/utils/request/index.ts

// 后台请求实例（baseURL: /admin）
export const adminRequest = createRequest('/admin')

// 前台请求实例（baseURL: /site）
export const siteRequest = createRequest('/site')

// 默认导出后台请求
export default adminRequest
```

改造步骤：
- [ ] 修改 `src/utils/request/index.ts`
- [ ] 导出 `adminRequest` 和 `siteRequest`
- [ ] 更新 API 文件使用对应请求实例

## 3. 路径别名更新

```typescript
// vite.config.ts
alias: {
  '@': resolvePath('src'),
  '@api': resolvePath('src/api'),
  '@views': resolvePath('src/views'),
  '@backend': resolvePath('src/views/backend'),
  '@frontend': resolvePath('src/views/frontend'),
}

// tsconfig.json
paths: {
  "@/*": ["src/*"],
  "@api/*": ["src/api/*"],
  "@views/*": ["src/views/*"],
  "@backend/*": ["src/views/backend/*"],
  "@frontend/*": ["src/views/frontend/*"]
}
```

## 4. 插件管理器

```
新增：
  src/utils/plugin/
    ├── PluginManager.ts   # 插件管理器
    ├── types.ts           # 类型定义
    └── index.ts           # 导出
```

功能：
- [ ] 读取插件列表
- [ ] 安装插件（复制文件）
- [ ] 卸载插件（删除文件 + 备份）
- [ ] 启用/禁用插件
- [ ] 插件状态同步

## 5. 菜单与权限

- [ ] 菜单数据增加 `pluginId` 字段标识来源
- [ ] 权限点支持插件前缀 `plugin:wechat:xxx`
- [ ] 插件卸载时自动清理相关菜单和权限

## 6. 执行优先级

1. **第一步**：目录结构调整（API + Views）
2. **第二步**：路由拆分
3. **第三步**：Request 封装
4. **第四步**：Store 拆分
5. **第五步**：插件管理器
6. **第六步**：插件管理界面
